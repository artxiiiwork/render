# RENDER — заметки по проекту

## Что это
RENDER — **доска вакансий и резюме** для видеомонтажёров («hh.ru для монтажёров»
под YouTube, Shorts, Reels, TikTok). Работодатели (блогеры, студии, агентства,
бренды) размещают вакансии и ищут монтажёров в каталоге; монтажёры публикуют
резюме с шоурилом и откликаются.

**Это НЕ биржа.** Площадка НЕ проводит деньги между сторонами: нет заказов,
эскроу, платежей, споров и выплат. Только поиск и контакт. (Отдельный проект
`video-birzha` рядом — это именно биржа, его сюда не смешиваем.)

## Стек
- Next.js 16 (App Router, TypeScript, src/), React 19
- Tailwind CSS v4
- Prisma 6.19.3 + PostgreSQL (база — **отдельная** от биржи)
- Авторизация — next-auth v5 + bcryptjs (готово): `src/auth.ts`, типы в
  `src/types/next-auth.d.ts`, регистрация `src/app/api/register/route.ts`,
  формы `src/app/register` и `src/app/login`, кабинет `src/app/dashboard`.
  Сессия в JWT-cookie, роль и id прокинуты в session. Ключ — `AUTH_SECRET` в `.env`.
- Заполнение резюме/профиля (готово): `src/app/profile/edit` (server-страница +
  формы `EditEditorForm`/`EditEmployerForm`, сохранение через server actions в
  `actions.ts`). Подписи справочников — `src/lib/labels.ts`.
  Примечание: профили создаются со статусом модерации PENDING; гейт каталога по
  модерации ещё не сделан — решить при сборке каталога монтажёров.
- Каталог монтажёров (готово): `src/app/editors` — список с фильтрами (формат,
  поиск по имени/специализации/описанию, доступность, удалёнка) + пагинация
  (`src/components/Pagination.tsx`), карточки с `src/components/Avatar.tsx`.
  Страница резюме `src/app/editors/[id]` оформлена в Behance-стиле: обложка-шапка
  во всю ширину (поле `EditorProfile.coverUrl`, загрузка через `ImageUpload`; если
  нет — градиентная заглушка), аватар поверх обложки. Слева узкая колонка: имя,
  статус, «Связаться», блок «о себе» (`ProfileBio.tsx` — сворачивает длинный текст)
  и кнопка «Подробнее», открывающая модалку `ResumeDetails.tsx` со всеми
  характеристиками (теги-чипсы + факты с иконками). Справа — большая галерея
  роликов сеткой (`PortfolioGallery.tsx`). Владелец резюме (сверяем `editor.userId`
  с сессией) видит карточку-плюс `AddPortfolioCard.tsx` (добавить ссылку +
  описание), может перетаскивать ролики мышью для смены порядка (поле
  `PortfolioLink.position`, сортировка по нему), а по карандашу на ролике —
  редактировать или удалить его. Server actions в `[id]/actions.ts`:
  `addPortfolioLink`, `updatePortfolioLink`, `deletePortfolioLink`,
  `reorderPortfolio` (все с проверкой владельца). В кабинет монтажёра добавлена
  кнопка «Посмотреть моё резюме». Блок «о себе» владелец правит на месте по
  карандашу (`ProfileBio.tsx` → action `updateBio`). Встроенный шоурил
  (`src/lib/embed.ts` распознаёт YouTube/Vimeo). Кнопка «Связаться»
  (`ContactButton.tsx`) открывает окно первого сообщения; отправка через action
  `messageUser` создаёт диалог и ведёт в чат (`RevealContact.tsx` больше не
  используется — раньше показывал email). Каталог требует входа,
  премодерации нет (показываем все резюме — решено на старте ради массы). Ролики в
  портфолио — только ссылки (не загрузка файлов: видео тяжёлое/дорогое, загрузку
  файлов отложили до трафика).
- Вакансии и отклики (готово): `src/app/vacancies` — создание/редактирование
  (`VacancyForm` + `actions.ts`), каталог с фильтрами + пагинация, страница
  вакансии `[id]` (отклик монтажёра `ApplyForm`, управление откликами
  `ApplicationActions`, открыть/закрыть `VacancyStatusButton`). Кабинет
  (`src/app/dashboard`) показывает «мои вакансии» (работодатель) и «мои отклики»
  (монтажёр). Сейчас показываем вакансии без премодерации (как и резюме).
- Заявки + личные сообщения (готово): страница `src/app/applications` (только
  работодатель) — список откликов по всем своим вакансиям; клик открывает
  модальное окно `ApplicationsBoard.tsx` (аватар+ник, сообщение, ролики
  портфолио, кнопки «Написать»/«Пригласить»/«Отказать»). «Написать» создаёт
  переписку (`openConversationWith`) и ведёт в чат. Чат: модели `Conversation` +
  `Message`. Раздел `src/app/messages` сделан в стиле мессенджера: общий
  `layout.tsx` (слева список переписок `ConversationList.tsx` с поиском +
  подсветкой активного, справа — панель чата), `page.tsx` — заглушка «выберите
  переписку», `messages/[id]/page.tsx` — сам чат + `MessageComposer.tsx`. Действия
  в `src/app/messages/actions.ts` (`openConversationWith`, `messageUser`,
  `sendMessage`). Чат — между работодателем и монтажёром, одна нить на пару.
- Уведомления о новом (готово): поля `Conversation.employerLastReadAt` /
  `editorLastReadAt` хранят, когда сторона читала переписку. Подсчёт —
  `src/lib/unread.ts` (`getUnreadConversationIds`, `countNewApplications`). В
  кабинете бейджи у «Сообщения» и «Заявки», в списке переписок — точка у
  непрочитанных. Чат отмечается прочитанным при открытии (`MarkRead.tsx` →
  `markConversationRead`); открытие отклика в `ApplicationsBoard` ставит ему статус
  VIEWED (счётчик новых откликов уменьшается).
- Админка/модерация (готово): страница `src/app/admin` доступна только при
  `User.isAdmin` (проверяем запросом к базе, в сессии флага нет). Сводка + списки
  всех резюме и вакансий с удалением (`AdminDeleteButton.tsx` →
  `deleteEditorProfileAdmin` / `deleteVacancyAdmin` в `admin/actions.ts`, каскадом
  чистятся ролики/отклики). При удалении админ указывает причину в окне — она
  уходит пользователю сообщением от службы поддержки. Премодерации нет — это
  инструмент чистки спама. Ссылка «Админка» в шапке кабинета видна только админу.
  Назначить админа: `node scripts/grant-admin.mjs <email>` (есть и
  `scripts/list-users.mjs`).
- Канал поддержки (готово): системный аккаунт «Поддержка RENDER»
  (`support@render.system`, создаётся автоматически при первом действии, войти
  нельзя). Переписка помечается `Conversation.isSupport=true` (поддержка — в слоте
  employer, пользователь — в editor). В мессенджере такой чат выделен: иконка-щит,
  бейдж «Поддержка»/«Официально». Сообщения шлёт `sendSupportMessage` внутри
  `admin/actions.ts` (пока односторонне — уведомления от модерации; ответы
  пользователя уходят на аккаунт поддержки, которого никто не читает).
  ⚠ База Neon (free) засыпает → в `src/lib/prisma.ts` добавлен авто-повтор запроса
  при ошибке P1001 (до 3 попыток), чтобы холодный старт не выдавал 500.
- Загрузка картинок (аватар/логотип): компонент `src/components/ImageUpload.tsx`
  + маршрут `src/app/api/upload/route.ts`. Сейчас файлы сохраняются локально в
  `public/uploads` (в git не попадают). Ближе к запуску заменить на облако
  (Яндекс Object Storage, как в бирже) — менять только `route.ts`, поля в базе
  (`avatarUrl`/`logoUrl`) и форма останутся прежними.

## Дизайн
- **Тёмная тема** по умолчанию.
- Акцент — перивинкль из логотипа After Effects: `#9B8CFF` (ховер `#B5A8FF`).
- Референс по духу: 8bitbrain.store — премиальный минимализм, воздух, карточный
  каталог, высококонтрастные кнопки.
- Все цвета вынесены в дизайн-токены в `src/app/globals.css` — менять там.
- Шрифты (подключены в `src/app/layout.tsx` через next/font, оба с кириллицей):
  - заголовки и логотип — **Montserrat** жирных начертаний (700/800/900),
    «плакатное» начертание; токен `--font-display`, утилита `font-display`;
  - основной текст — **Inter**; токен `--font-sans` (используется в `body`).
- Премиальные приёмы (из референса 8bitbrain): плашки `.panel` с внутренним
  градиентом и свечением, кнопки-таблетки `.btn-accent`, надзаголовки `.eyebrow`,
  мягкое фоновое свечение на `body` — всё в `src/app/globals.css`.
- ⚠ Засада сборщика (Tailwind v4 / Lightning CSS): НЕ использовать многослойный
  `background:` шорткат со сложным `radial-gradient(... at ... -5% ...)` — парсер
  молча роняет правило и весь CSS после него. Использовать раздельно
  `background-color` + `background-image` с простым `radial-gradient(circle at ...)`.

## Модель данных
См. `prisma/schema.prisma`. Основные карточки:
- `User` — общий аккаунт (роль EDITOR | EMPLOYER, флаг isAdmin).
- `EditorProfile` — резюме монтажёра + `PortfolioLink` (ссылки на шоурил).
- `EmployerProfile` — профиль работодателя.
- `Vacancy` — вакансия (с заделом под платное продвижение: `isPromoted`).
- `Application` — отклик монтажёра на вакансию.

## Как работаем
Владелец — не программист. Объяснять простыми словами, идти небольшими шагами,
спрашивать на развилках. На старте всё бесплатно; модель данных подготовлена под
платные функции (продвижение вакансий, доступ к базе резюме, подписка) без
переделки.
